## まえがき

Windows で LaTeX の環境を構築するのはとにかくべらぼうに時間がかかることが知られています．インストールに 1 日近くかかるというのは決して大げさな話ではありません．

下の表は，以前私が GitHub Actions を用いて TeX Live のインストールにかかる時間を実際に計測したものです．

| 環境    |  1 回目 |  2 回目 |
| ------- | ------: | ------: |
| Windows | 48m 15s | 46m 13s |
| Linux   |  8m 40s |  6m 11s |
| Docker  |  1m 28s |  1m 33s |

https://zenn.dev/fjktkm/articles/f43658e74f5814

Docker を使うと，Windows に直接インストールするのと比べておよそ 30 倍速く環境構築が完了することがわかります．これは GitHub Actions のランナーで計測した結果なので，多少結果は変化するかもしれませんが，ともかく Windows に直接インストールするより Docker を使った方が圧倒的に速いことは間違いありません．

本 Zenn Book では，Dev Container を用いて LaTeX 環境を構築する方法を解説します．Dev Container を使うことで，Windows に直接インストールするよりはるかに短い時間で LaTeX 環境を構築できます．本 Zenn Book を通じて，LaTeX の環境構築で挫折する人間が一人でも減ることを願っています．

## 想定読者

想定している読者は次のような人です．

- LaTeX の環境構築にうんざりしている人

はじめて Dev Container に触れる人でも理解できるように，手順を懇切丁寧に解説しています．ぜひこの機会に Dev Container を使った環境構築に挑戦してみてください．

## 注意事項

本 Zenn Book で紹介する方法は，2026 年 4 月時点の情報に基づいています．技術の進歩やツールの更新により，内容が古くなる可能性があります．また，環境構築の手順は個々の PC の設定によって異なる場合があります．もし問題が発生した場合は，適宜生成 AI を活用するなどしてください．
