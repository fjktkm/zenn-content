## 仮想化とは

仮想化とは，物理的なコンピュータの計算リソースをソフトウェア的に分割し，あたかも複数のコンピュータが存在するかのように利用する技術です．仮想化には仮想マシンとコンテナ型の 2 種類があります．

仮想マシンは，物理的なコンピュータの中に完全な仮想的なコンピュータを作る技術です．OS も含めて完全に独立した環境を作ることができる一方で，オーバーヘッドが大きいなどのデメリットがあります．

一方で，コンテナ型の仮想化は，OS はホストマシンと共有しつつアプリケーションの実行環境だけを仮想化する技術です．オーバーヘッドが小さく高速に動作するため，現代ではソフトウェア開発の現場で広く利用されています．

## Docker とは

Docker は，コンテナ型の仮想化技術のひとつです．コンテナ型の仮想化技術のデファクトスタンダードであり，世界中の開発者に利用されています．

なお，コンテナは基本的に Linux 上で動作します．Windows 上で直接動かすことはできないため，Windows でコンテナを動かしたい場合は WSL を活用する必要があります．前の章で WSL をセットアップしたのは，Docker を利用するための準備だったというわけです．

## Docker Desktop のインストール

Docker Desktop をインストールします．Docker Desktop がすでにインストール済みの場合は，このセクションを読み飛ばしてください．推奨するインストール手順は次の通りです．

1. 以下のページを開き「Microsoft Store で見る」ボタンをクリックする
2. Microsoft Store アプリを起動するかどうかを確認するポップアップが表示された場合は承認する
3. Microsoft Store アプリの Docker Desktop のページが開いたら，「インストール」ボタンをクリックしてインストールを実行する
4. ユーザーアカウント制御のダイアログが表示された場合は「はい」をクリックして承認する

https://apps.microsoft.com/detail/XP8CBJ40XLBWKX?hl=ja&gl=JP&ocid=pdpshare

## Docker Desktop の起動

インストールが完了したら，次に Docker Desktop を起動します．推奨する起動手順は次の通りです．

1. Windows キーを押してスタートメニューを開く
2. 検索バーに「docker desktop」と入力する
3. 検索結果の「Docker Desktop」の項目をクリックして起動する

## Docker Desktop の初期設定

Docker Desktop が起動したら，初回起動時は「Welcome to Docker」というタイトルのポップアップが表示されます．好きな方法でアカウントを作成またはログインしてください．スキップしても問題ありません．

ダッシュボードの左下に「Engine running」と表示されていれば Docker が正常に起動しています．問題が発生している場合は，以下の公式ドキュメントのページを参考にしてください．

https://docs.docker.com/desktop/

以上で Docker のセットアップは完了です．Docker Desktop のウィンドウは閉じても Docker はバックグラウンドで動作し続けるので，Docker Desktop のウィンドウは閉じてしまって構いません．
