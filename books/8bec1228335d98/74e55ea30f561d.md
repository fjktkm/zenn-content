## TeX Live 公式イメージ

TeX Live は世界でもっとも広く使われている TeX ディストリビューションです．TeX Live プロジェクトは公式の Docker イメージ `texlive/texlive` を提供しています．

https://hub.docker.com/r/texlive/texlive

## `devcontainer.json` の編集

前のチャプターで作成した `devcontainer.json` を次のように書き換えてください．

```json
{
  "name": "LaTeX (TeX Live)",
  "image": "texlive/texlive:latest",
  "features": {
    "ghcr.io/devcontainers/features/common-utils:2": {}
  }
}
```

各項目について簡単に説明します．

`name` は Dev Container の名前で，任意の文字列を指定できます．ここでは「LaTeX (TeX Live)」としましたが，プロジェクトの内容に応じて適切な名前をつけてください．

`image` は Dev Container のベースイメージを指定する項目です．ここでは TeX Live の公式イメージである `texlive/texlive:latest` を指定します．

`features` は Dev Container Feature を設定する項目です．Dev Container Feature とは，`image` に指定したイメージに含まれていないツール等を追加するための仕組みです．これを利用することで，イメージを簡単にカスタマイズできます．

ここでは `ghcr.io/devcontainers/features/common-utils:2` を指定しています．`common-utils` は開発によく用いられるコマンドラインツールやシェルの設定を追加するための Dev Container Feature です．Microsoft 謹製イメージにはあらかじめ追加されています．`texlive/texlive` など Microsoft 謹製でないイメージをベースにする場合は，`common-utils` を追加すると謹製イメージと同じ使い勝手になり便利です．

## コンテナのリビルド

構成ファイルの変更はコンテナをリビルドしないと反映されません．構成ファイルを編集したら，コンテナをリビルドします．推奨する手順は次の通りです．

1. VS Code のウィンドウの左下にある「><」のアイコンをクリックする
2. 表示されたメニューの中から「コンテナーのリビルド」を選択する

初回実行時は `texlive/texlive` イメージのダウンロードが行われるため，リビルドにはすこし時間がかかります．

## LaTeX の動作確認

コンテナのリビルドが完了したら，LaTeX が利用できるようになっているか確認します．ターミナルで以下のコマンドを実行してください．

```bash
lualatex --version
```

以下のような内容が表示されれば成功です．

```bash
$ lualatex --version
```

## TeX ファイルの作成

LaTeX が利用できることが確認できたら，次は TeX ファイルを作成します．プロジェクトフォルダに `main.tex` というファイルを作成してください．ファイルの中身は次のようにしてください．

```latex
\documentclass{jlreq}

\begin{document}

\title{ポラーノの広場}
\author{宮沢賢治}
\maketitle

\section{はじめに}

あのイーハトーヴォのすきとおった風、夏でも底に冷たさをもつ青いそら、うつくしい森で飾られたモリーオ市、郊外のぎらぎらひかる草の波。

\end{document}
```

ドキュメントクラスには `jlreq` を使用します．`jlreq` は日本語の組版規則に準拠した最新のドキュメントクラスです．現在新規で日本語文書を作る場合は `jlreq` を使うことを推奨します．

## TeX ファイルのコンパイル

TeX ファイルを作成したら，次はコンパイルして PDF を生成します．ターミナルで以下のコマンドを実行してください．

```bash
lualatex main.tex
```

しばらくするとコンパイルが完了し，`main.pdf` が生成されます．エクスプローラーから `main.pdf` を開いて中身を確認してください．

以上で LaTeX イメージの導入は完了です．
