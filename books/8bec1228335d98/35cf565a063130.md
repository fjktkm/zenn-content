## full イメージのサイズについて

これまでのチャプターでは `texlive/texlive:latest` イメージを使ってきました．このイメージは TeX Live の full スキームをベースとしており，ほぼすべての LaTeX パッケージが含まれているため，追加でパッケージをインストールする手間がかかりません．

一方で，圧縮サイズが約 2.5 GB とやや大きめなため，ストレージ容量やダウンロード量をさらに抑えたい場合もあるかもしれません．そのような場合には，より小さいスキームのイメージを使い，不足しているパッケージだけを `tlmgr` で追加する方法があります．

## イメージのサイズ比較

`texlive/texlive` イメージには，TeX Live のスキームに対応した複数のタグが用意されています．

| タグ             | スキーム | 圧縮サイズ |
| ---------------- | -------- | ---------: |
| `latest`         | full     |  約 2.5 GB |
| `latest-medium`  | medium   |  約 900 MB |
| `latest-small`   | small    |  約 600 MB |
| `latest-basic`   | basic    |  約 370 MB |
| `latest-minimal` | minimal  |  約 330 MB |

スキームが小さくなるほど含まれるパッケージの数が少なくなります．`minimal` は TeX のコアのみを含む最小構成で，`full` はすべてのパッケージを含む最大構成です．

`latest` より小さいスキームのイメージには，日本語組版に必要なパッケージが含まれていません．日本語を正しく組版するためには `tlmgr` でパッケージを別途追加する必要があります．ベースイメージには `latest-small` が扱いやすいでしょう．LuaLaTeX エンジンや Latexmk など，LaTeX 環境として最低限必要なツールが一通り含まれており，サイズと利便性のバランスが取れています．

## `tlmgr` とは

`tlmgr` は TeX Live のパッケージマネージャです．コマンドラインからパッケージのインストールや更新を行うことができます．パッケージをインストールするには，次のコマンドを実行します．

```bash
tlmgr install <パッケージ名>
```

Dev Container では，`devcontainer.json` の `postCreateCommand` を使って，コンテナ作成時にこのコマンドを自動で実行するよう設定できます．日本語文書をコンパイルする場合は，`collection-langjapanese` を追加するとよいでしょう．ただし，これもサイズが約 334 MB 程度あるため，気になる場合はインストールするパッケージをさらに絞り込むこともできます．

## `devcontainer.json` の設定例

`texlive/texlive:latest-small` をベースイメージとして使用し，`postCreateCommand` で `collection-langjapanese` をインストールする設定例を次に示します．

```diff jsonc:.devcontainer/devcontainer.json
 {
   "name": "LaTeX (TeX Live)",
-  "image": "texlive/texlive:latest",
+  "image": "texlive/texlive:latest-small",
   "mounts": [
     {
       "target": "${containerWorkspaceFolder}/build",
       "type": "volume"
     }
   ],
   "features": {
     "ghcr.io/devcontainers/features/common-utils:2": {}
   },
   "customizations": {
     "vscode": {
       "extensions": ["James-Yu.latex-workshop"]
     }
   },
   "onCreateCommand": "sudo chown -R vscode:vscode build",
+  "postCreateCommand": "sudo tlmgr install collection-langjapanese",
   "remoteUser": "vscode"
 }
```

`postCreateCommand` に指定したコマンドはコンテナの作成後に自動で実行されます．はじめに `tlmgr update --self` で `tlmgr` 自体を最新版に更新し，続けて `tlmgr install` で必要なパッケージをインストールしています．

`devcontainer.json` の編集が完了したら，コンテナをリビルドしてください．

## セットアップ時間の比較

以下は筆者の環境での初回セットアップ時間の計測結果です（ネットワーク環境により異なります）．

| 方法                     | 初回セットアップ |
| ------------------------ | ---------------: |
| `latest` (full)          |        約 432 秒 |
| `latest-small` + `tlmgr` |        約 122 秒 |

`latest-small` + `tlmgr` の方が初回セットアップを大幅に短縮できます．ただし，`latest` は初回の Pull に時間がかかるだけで，以降は Docker のキャッシュが効きます．また，`latest-small` + `tlmgr` のようにパッケージを自分で管理する手間もかかりません．ストレージや通信量が気にならなければ `latest` タグをそのまま使うとよいでしょう．

以上でイメージサイズの最適化は完了です．
