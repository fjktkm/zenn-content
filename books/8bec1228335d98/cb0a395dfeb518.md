title: "GitHub Actions による CI/CD の構築"
---

## CI/CD とは

CI/CD とは，継続的インテグレーション（Continuous Integration）と継続的デリバリー/デプロイメント（Continuous Delivery/Deployment）の略で，ソフトウェア開発のプロセスを自動化するための手法やツールの総称です．コードの変更がリポジトリにプッシュされるたびに，自動でビルドやテスト，デプロイなどの一連の作業が実行される仕組みを指します．

## GitHub Actions とは

GitHub Actions は GitHub が提供する CI/CD ツールです．GitHub と高度に統合されており，GitHub Pages や GitHub Release などの機能と組み合わせて使うことができます．ワークフローファイルをリポジトリ内に配置するだけで簡単に CI/CD を構築できます．

LaTeX プロジェクトにおける CI/CD には様々なユースケースが考えられますが，ここでは GitHub Actions を使ってリリース用 PDF を自動生成し GitHub Release に公開するワークフローを構築します．

## PDF の出力先の変更

PDF を画像として挿入する場合，ワークフローの中で画像素材の PDF とビルド結果の PDF が混ざってしまい，リリース用 PDF の収集が煩雑になることがあります．リリース用 PDF を収集しやすくするため，ビルドした PDF の出力先を変更しておくと便利です．

ビルドした PDF の出力先を変更するには， `.latexmkrc` に `$out_dir` の設定を追加します．プロジェクトフォルダにある `.latexmkrc` を以下のように書き換えてください．

```perl
$out_dir = 'dist';

$pdf_mode = 4;

$lualatex = 'lualatex -synctex=1 -interaction=nonstopmode %O %S';
$bibtex = 'upbibtex %O %B';
```

この変更によりビルドされた PDF が `dist` フォルダに出力されるようになります．ワークフローでリリース用 PDF を収集する際は，この `dist` フォルダ内にある PDF を収集すればよいことになります．

## VS Code の PDF プレビューの設定の変更

PDF の出力先の変更が完了したら，次は VS Code の PDF プレビューの設定を変更します．VS Code の PDF プレビューはデフォルトで同階層の PDF を探すようになっているため，PDF の出力先を変更した場合は `settings.json` にも対応する設定を追加する必要があります．以下の内容に `settings.json` を書き換えてください．

```json
{
  "files.associations": {
    ".latexmkrc": "perl"
  },
  "latex-workshop.latex.outDir": "dist",
  "latex-workshop.latex.recipe.default": "latexmk (latexmkrc)",
  "latex-workshop.formatting.latex": "latexindent"
}
```

## ワークフローファイルの作成

`.github/workflows/release.yml` を作成してください．ファイルの中身は次のようにしてください．

```yaml
name: release

on:
  workflow_dispatch:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest
    container:
      image: texlive/texlive:latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v6

      - name: Compile LaTeX
        run: find . -name .latexmkrc -execdir latexmk \;

      - name: Upload PDFs artifact
        uses: actions/upload-artifact@v7
        with:
          name: pdfs
          path: dist/*.pdf

  release:
    needs: build
    runs-on: ubuntu-latest
    permissions:
      contents: write

    steps:
      - name: Checkout repository
        uses: actions/checkout@v6

      - name: Download PDF artifact
        uses: actions/download-artifact@v8
        with:
          name: pdfs
          path: .

      - name: Create GitHub Release
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          tag="v${{ github.run_number }}"
          gh release create "$tag" ./*.pdf --title "$tag" --generate-notes
```

## リポジトリを GitHub に発行する

ワークフローファイルの作成が完了したら，次はリポジトリを GitHub に発行します．推奨する手順は以下の通りです．

1. VS Code のウィンドウの下部にある「GitHub に発行する」ボタンをクリックする
2. GitHub へのサインインを求められたら，GitHub アカウントでサインインする
3. 「Publish to GitHub private repository」または「Publish to GitHub public repository」を選択する

GitHub にリポジトリが発行されたら，ワークフローの動作確認を行います．これまでの変更をコミットしてプッシュしてください．プッシュしてしばらくすると GitHub Actions が自動で走りビルドされた PDF が Release に公開されます．

以上で GitHub Actions を使った CI/CD の構築は完了です．
